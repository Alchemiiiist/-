1. 使用 1_trimmomatic.sh 去除测序接头暨进行质量控制。
	--仅需修改find中的文件路径即可
	--使用该代码完成一次去接头操作可能需要3min左右，大数据量情况下可使用screen将该过程放入后台


2. 使用 2_fastqc.sh 查看质控前后文库质量。
	--直接调用该代码即可
	--建议使用screen将该过程放入后台


3. 使用 3_hisat2.sh 进行测序结果比对，得到sam和排序后的bam文件。
	--初次使用时，修改好索引文件夹路径，一般情况直接调用即可，代码使用的是人类基因组的注释文件
	-- -x 是索引文件参数，输入要使用的索引文件所在的文件夹路径
	--注意索引是否是最新版，代码中使用的是GRCh38
	--注意使用的索引是否合适是对应的人类或相应的模式生物
	--索引文件可以在hisat2官网进行下载
	--hisat2非常适用短序列的比对，RNA-seq

（选）3. 使用 3_STAR.sh进行测序结果比对，得到sam和排序后的bam文件
	--与 3_hisat2.sh 选择使用一种即可。
	--初次使用时，提前构建好索引文件，修改索引文件存储位置与基因注释文件路径
	--需要将所有trim后的*P.fq.gz文件解压为*.fq文件后，再直接调用。建议重新安排文件夹进行储存
	--star适合短序列和较长序列的比对。
	--该代码的注释部分提供了索引构建的代码，在第一次使用时，请先使用此部分代码构建索引
	--注意注释文件的选择，本代码使用gencode 的注释文件

4. 使用 4_htseq.sh 得到表达矩阵
	--初次使用时，修改基因注释文件路径与salmon索引文件路径，一般情况直接调用即可
		--可以由htseq-count得到count原始表达矩阵--用于后续差异分析。（也可以手动转化成TPM，FPKM表达矩阵）
		--注意htseq-count使用的gtf基因注释文件的版本与类型是否合适，gtf注释文件可以在ensembl上下载。
		--（推荐hisat2流程使用ensembl的注释文件，这两者染色体命名一致；gencode上下载的gtf注释文件与hisat2得到的sam文件染色体命名不一致的问题还没有解决）
		--本代码使用的union模型判断reads属于哪一个基因，大多数情况推荐使用union

		--可以由salmon quant得到定量结果，并进一步获取TPM表达矩阵
		--不研究新的转录本，研究RNA-seq过程基因表达发生了何种变化，使用salmon方法可以更方便更快得基因定量信息
		--提前构建好salmon index（使用--gencode参数，-t后接gencode下载的转录本（cDNA）注释文件；不适用该参数，-t后接ensembl的转录本gtf）
		--本代码中使用的是gencode的注释文件，kmer取默认值。、
		--索引问价-i 输入提前构建好的索引文件夹目录
		--注释文件-g 建议与构建索引文件时使用的注释文件来源一致（gencode/ensembl）--提供转录本与基因间的关系，提供该参数，会额外输出quant.genes.sf，提供基因表达水平的估计
		--本代码输出基因表达量>1的tpm表达矩阵
（选）4. 使用4_htseq_star.sh 得到表达矩阵
	-- 与htseq.sh 几乎完全相同，部分文件夹做了修改，与STAR.sh结合使用即可，注意事项同4_htseq.sh
